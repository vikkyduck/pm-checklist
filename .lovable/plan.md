

## План: подключить Яндекс 360 для бизнеса как SMTP для магических ссылок

### Цель
Магические ссылки для входа PM Авито будут уходить с вашего корпоративного Яндекс-ящика (например, `noreply@вашдомен.ru`) вместо дефолтного адреса Supabase. Это улучшит доставляемость и брендирование.

### Что вам нужно подготовить (5 минут в Яндекс 360)

1. **Создать пароль приложения** для SMTP (основной пароль от ящика не подойдёт):
   - Зайдите на [id.yandex.ru](https://id.yandex.ru) под учёткой ящика
   - Безопасность → Пароли приложений → «Создать новый пароль»
   - Тип: **Почта** → название: `Lovable PM Checklist`
   - Скопируйте сгенерированный пароль (16 символов) — он показывается один раз

2. **Собрать 6 параметров SMTP**:
   | Параметр | Значение |
   |---|---|
   | Host | `smtp.yandex.ru` |
   | Port | `465` |
   | User | полный email, например `noreply@вашдомен.ru` |
   | Password | пароль приложения из шага 1 |
   | Sender email | тот же email (`noreply@вашдомен.ru`) |
   | Sender name | `Чек-лист PM Авито` |

### Что я сделаю в Lovable Cloud

1. **Открою для вас защищённую форму настроек SMTP** в Lovable Cloud → Users → Auth Settings → SMTP. Там есть встроенное поле для всех 6 параметров — пароль вы вводите туда напрямую, в чат присылать не нужно.

2. **Настрою rate limit** на стороне Supabase: Яндекс 360 для бизнеса даёт лимит ~150 писем в час с одного ящика. Этого хватит для пилота PM-команды Авито с запасом.

3. **Обновлю шаблон письма магической ссылки** — добавлю русский текст с брендингом «Чек-лист PM Авито» вместо дефолтного английского письма Supabase. Это делается через Auth Email Templates в настройках.

4. **Проверю TTL ссылки**: сейчас стоит дефолтный 1 час — оставлю как есть, это безопасно и удобно.

### Что НЕ меняется

- База данных остаётся на инфраструктуре Lovable Cloud (вы выбрали «только канал писем» по 152-ФЗ)
- Логика входа в `src/routes/login.tsx` и `src/routes/auth.callback.tsx` остаётся прежней
- Прогресс чек-листа продолжает сохраняться через `useChecklistProgress` без изменений

### Технические детали

- SMTP конфигурируется на уровне Supabase Auth (GoTrue), не в коде приложения — поэтому не будет правок в репозитории
- Письма по 152-ФЗ: транзит письма пойдёт через серверы Яндекса в РФ (ФЗ-152 это допускает, поскольку пересылка через российский почтовый сервер считается локализованной обработкой)
- Если ящик заблокирует SMTP за подозрительную активность — переключиться обратно на дефолт можно одной кнопкой в той же форме
