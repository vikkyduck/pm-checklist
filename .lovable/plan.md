

## Боковое меню навигации (dev-only)

Добавлю слева сворачиваемый сайдбар со списком всех страниц проекта, чтобы быстро переключаться между ними в превью.

### Что появится

Сайдбар с разделами:

**Публичные**
- `/` — Главная (чек-лист)
- `/login` — Вход
- `/negotiations` — Переговоры

**Только для авторизованных**
- `/admin` — Админка (с пометкой «admin»)
- `/auth/callback` — Auth callback (редирект, для отладки)

**Email-превью (dev)**
- `/lovable/email/auth/preview` — превью шаблонов писем

Текущий маршрут подсвечивается. Сайдбар сворачивается в узкую полоску с иконками через кнопку-триггер в шапке.

### Технические детали

1. **`src/components/dev-nav-sidebar.tsx`** (новый) — компонент `Sidebar` из shadcn с группами `SidebarGroup` → `SidebarMenu` → `SidebarMenuItem`. Каждый пункт — `<Link to="...">` из `@tanstack/react-router` с `activeProps` для подсветки. Использует `collapsible="icon"`, чтобы при сворачивании оставались иконки (Home, LogIn, MessageSquare, Shield, Mail).

2. **`src/routes/__root.tsx`** — оборачиваю содержимое в `SidebarProvider` с `defaultOpen={false}` (чтобы не мешал обычному просмотру), рядом с `<Outlet />` ставлю `<DevNavSidebar />` и плавающий `<SidebarTrigger />` в фиксированной позиции `top-3 left-3 z-50`. Триггер всегда виден, по клику выезжает меню.

3. **Видимость только в dev/preview**: оборачиваю в условие `import.meta.env.DEV || window.location.hostname.includes('lovable.app')` — на продакшен-домене (`pm-checklist.lovable.app` если он будет на своём домене) меню не появится, но в preview-домене Lovable будет работать.

4. **Стилизация**: использую существующие токены темы (`bg-card`, `text-foreground`, акцент `--stage-*`), чтобы сайдбар вписался в визуал платформы. Ширина в развёрнутом виде — стандартная (`w-64`), в свёрнутом — `w-14`.

5. Шапки на `/` и `/negotiations` уже содержат свою навигацию — сайдбар их не заменяет, а дополняет (для быстрого dev-переключения).
