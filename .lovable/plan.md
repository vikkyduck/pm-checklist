

## План: вход по магической ссылке + сохранение прогресса для PM Авито

Поскольку у вас нет готового API на Beget/Timeweb, я подготовлю **полное решение под ключ**: фронт + бэкенд-код + инструкция по развёртыванию на вашем хостинге.

---

### Что получите в итоге

1. **Бэкенд** (Node.js + Express + PostgreSQL + nodemailer) — готовый код, который вы загрузите на Beget/Timeweb через FTP/SSH.
2. **Email-уведомления** — письма с магической ссылкой будут уходить через SMTP вашего хостинга (Beget/Timeweb предоставляют SMTP бесплатно).
3. **Фронт** — страница входа, callback-страница, синхронизация прогресса чек-листа с вашим API.
4. **Инструкция по развёртыванию** — пошаговая, с командами и скриншотами разделов панели Beget/Timeweb.

### Бэкенд: эндпоинты

| Метод | Путь | Назначение |
|---|---|---|
| POST | `/auth/magic-link` | Принимает `{email, name}`, генерирует токен (TTL 15 мин), отправляет письмо |
| GET | `/auth/verify?token=...` | Проверяет токен, возвращает JWT (срок 30 дней) |
| GET | `/checklist` | Возвращает прогресс PM (требует JWT) |
| PUT | `/checklist` | Сохраняет состояние галочек (требует JWT) |
| GET | `/me` | Профиль PM (email + имя) |

**Таблицы PostgreSQL:**
- `users` — id, email, name, created_at
- `magic_tokens` — token, user_id, expires_at, used_at
- `checklist_progress` — user_id, item_id (string), checked (bool), updated_at

### Фронт: что добавлю

1. **`/login`** — форма с полями Email + Имя, кнопка «Получить ссылку на почту». Glass-стилистика как в основной странице.
2. **`/auth/callback`** — экран «Входим…», обрабатывает `?token=...`, сохраняет JWT в `localStorage`, редиректит на `/`.
3. **Шапка `/`** — приветствие «Привет, {имя}» + кнопка «Выйти». Если не залогинен — редирект на `/login`.
4. **Чек-боксы у каждого пункта чек-листа** — галочка слева от заголовка `Item`. Состояние синхронизируется с API (debounce 500ms).
5. **Индикатор прогресса** в шапке каждого этапа: «3/8 выполнено».
6. **API-клиент** `src/lib/api.ts` — обёртка над `fetch` с авто-добавлением `Authorization: Bearer ...`. Базовый URL берётся из переменной `VITE_API_URL` (вы укажете после развёртывания).

### Как будет выглядеть процесс развёртывания (что вы будете делать)

**Шаг 1.** Я подготовлю папку `/backend` в проекте с готовым Node.js-приложением (`server.js`, `package.json`, `.env.example`, `schema.sql`, `README.md`).

**Шаг 2.** Вы скачаете эту папку (можно через GitHub-экспорт Lovable) и:
- В панели Beget/Timeweb создадите Node.js-приложение (один клик).
- Создадите PostgreSQL-базу (один клик), скопируете credentials.
- Загрузите файлы через FTP или Git.
- Заполните `.env` (DB credentials, SMTP credentials, JWT_SECRET, FRONTEND_URL).
- Выполните `npm install` и `node schema.sql` через веб-консоль хостинга.
- Запустите приложение.

**Шаг 3.** Скопируете URL вашего API (например, `https://pm-checklist.beget.tech`) и пришлёте мне — я добавлю его в `VITE_API_URL` фронта.

**Шаг 4.** Готово. PM вводит email на `/login` → получает письмо → кликает ссылку → входит → отмечает галочки → прогресс сохраняется.

### Технические детали (для разработчика, если будете передавать)

- **CORS**: бэк разрешит origin `https://pm-checklist.lovable.app` (и preview-домен).
- **Магический токен**: 32 байта `crypto.randomBytes`, hex, одноразовый, TTL 15 мин.
- **JWT**: `HS256`, секрет из `.env`, payload `{userId, email, name}`, exp 30 дней.
- **Email-шаблон**: HTML + plaintext, тема «Вход в чек-лист PM Авито», кнопка-ссылка `${FRONTEND_URL}/auth/callback?token=...`.
- **SMTP Beget**: `smtp.beget.com:465` (SSL), логин = email вашего домена.
- **SMTP Timeweb**: `smtp.timeweb.ru:465` (SSL).
- **Защита от спама**: rate-limit 1 запрос магической ссылки на email раз в 60 сек.
- **localStorage**: ключ `pm_checklist_jwt`, очищается при logout или 401-ответе.
